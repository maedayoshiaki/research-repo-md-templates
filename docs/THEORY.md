# THEORY.md — 数学・理論（人間所有）

研究の数式・導出・前提を記録する。**式は人間が所有する**（AGENTS.md の「Human review required」を参照）。
AI の役割は「整合性チェック・別解/文献提示・実装・検証」まで。**式の正しさの最終判断は人間**。

## Ownership Rule（所有ルール）

| 工程 | 担当 |
|------|------|
| 定式化（式・記号・前提を立てる） | 人間 |
| 整合性レビュー（次元・記号衝突・境界条件）／別解・文献の提示 | AI（提案のみ） |
| 正しさの最終判断 | 人間 |
| 実装（式↔コード対応） | AI（人間確認後） |
| 検証（解析解・極限・保存量） | AI+人間 |

- AI が生成・変形した式は **`（未確認）`** を付け、**人間レビュー前に実装しない**。
- 実装に入った式は必ず [Eq ↔ Code Map](#eq--code-map) に対応を書く。

## Workflow（式が実装に入るまで）

1. **人間が定式化** — 下の `Definitions` / `Derivations` に式・記号・前提を書く。
2. **AI が整合性レビュー** — 次元解析・記号の衝突・境界/初期条件・特異点の有無を指摘（修正案は出すが確定しない）。
3. **人間が確認・修正** — `（未確認）` を外す。
4. **実装** — コードに落とし、式番号と関数を対応づける。
5. **検証** — 既知の解析解・極限ケース・保存量・単位で単体テストを書く。
